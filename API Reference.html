neural_net.neural_net_parsing()¶

The neural_net_parsing function is designed to parse command-line arguments and execute a neural network training process based on the provided parameters. It reads a dataset from a file, preprocesses the data, creates and trains a neural network model using k-fold cross-validation, and optionally performs a grid search for hyperparameter optimization. As output the function prints the model structure

The parameters listed below are not parameters of the functions, but are parsing arguments to be used in terminal, when executing the program as follows:

$Your_PC>python neural_net.py file.csv --folds 7 --ex_cols 4 --plot

In order to read a description of every argument, execute:

$Your_PC>python neural_net.py --help
Parameters

  • filename (str) – name of the file that has to be analized

  • target (str) – optional (default = AGE_AT_SCAN): name of the colums holding target values

  • location (str) – optional: location of the file, i.e. folder containing it

  • hidden_layers (int) – optional (default = 2): number of hidden layers in the neural network

  • hidden_nodes (int) – optional (default = 48): number of hidden layer nodes in the neural network

  • epochs (int) – optional (default = 300): number of epochs of training

  • opt (str) – optional(default = “adam”): optimizer

  • folds (int) – optional (default = 5): number of folds in the k-folding

  • dropout (int) – optional (default = 0.01): dropout rate in neural network

  • ex_cols (int) – optional (default = 5): number of columns excluded when importing

  • summary (bool) – optional(default = False): show the summary of the neural network

  • plot (bool) – optional(default = False): show the plot of training history, actual vs predicted brain (TD and ASD)

  • group (str) – optional(default = ‘DX_GROUP’): name of the column indicating the group (experimental vs control)”)

  • overs (bool) – optional(default = False): oversampling, done in order to have a flat distribution of targets”)

  • harm (str) – optional: name of the column of sites, used for data harmonization

  • grid (bool) – optional(default = False): grid search for hyperparameter optimization



regression.regression(type, features, targets, n_splits)¶

performs regression (using sklearn) with k-fold cross-validation with a specified number of splits on the given dataset and prints evaluation metrics of the linear regression model such as MAE (mean absolute error) and R-squared. Regressions models implemented are linear and gaussian. The function inizialize a plot that shows actual vs. predicted age for a control group of patients.

Parameters

  • type (str) – type of regression performed, only two arguments are possible ‘linear’ or ‘gaussian’

  • features (numpy.ndarray) – features

  • targets (numpy.array) – array containing target feature

  • n_splits (int) – number of folds for cross-validation

Returns

A tuple containing:

  • best_model (sequential): the best model selected across k-folding.

  • mae (float): the mean absolute error mean across folds.

  • r2 (float): the coefficient of determination mean across folds.

  • pad_control (list): the predicted actual difference.

Return type

tuple(sequential, float, float, list)



regression.reg_parsing()¶

regression function parsed that runs when the .py file is called. It performs regression (linear or gaussian) with k-fold cross-validation predicting the age of patients from magnetic resonance imaging and prints evaluation metrics of the linear regression model such as MAE (mean absolute error) and R-squared. There are two ways to pass the csv file to this function. It’s possible to pass the absolutepath of the dataset or you can store the dataset in a brother folder of the one containing code, and pass to the parsing function the filename and his container-folder. The parameters listed below are not parameters of the functions but are parsing arguments that have to be passed to command line when executing the program as follow:

$Your_PC>python regression.py file.csv type --target --location --folds --ex_cols --plot

where file.csv and type are the only mandatory argument, while others are optional and takes some default values, that if they have to be modified you can write for example:

$Your_PC>python linear_reg.py file.csv --folds 10
Parameters

  • filename (str) – path to the CSV file containing the dataset or the name of the file if –location argument is passed

  • type (str) – type of regression performed, only two arguments are possible ‘linear’ or ‘gaussian’

  • target (str) – optional (default = AGE_AT_SCAN): name of the column holding target values

  • location (str) – optional: location of the file, i.e. folder containing it

  • folds (int) – optional (>4, default 5):number of folds for cross-validation

  • ex_cols (int) – optional (default = 5): columns excluded when importing file

  • overs (bool) – optional (default = False): if True oversampling on dataset is performed

  • plot (bool) – optional (default = False): show the plot of actual vs predicted brain age

  • group – optional (default = ‘DX_GROUP’): name of the column indicating the group (‘control’ vs ‘experimental’)

Returns

None
